关于草稿

‘’’hexo n draft “帖子名称”‘’’ 这是新建一个草稿
‘’’hexo s -d’’’ 这是在本地服务器让草稿显示出来，用于测试草稿
‘’’hexo p “帖子名称”‘’’ 这是发布一个草稿，或理解为升级为帖子
在网站级别 _config.yml中render_drafts的作用是控制 hexo n 命令是否生成draft下的md内容为html页面。
这是至关重要的开关，注意只能在本地预览时，打开，部署时一定不要忘记先关掉这个开关，以免发布草稿到你的远程网站！

关于服务器

‘’’hexo s’’’ 将会生成一个死站！就是你做了一些修改后，’’’hexo g’’’ 重新生成，但是refresh页面，没有任何变化！
而运行这个命令我们通常是要修改页面的，所以我们正常使用应该加个参数：’’’hexo s -s’’’ ，这样我们’’’hexo g’’’后可以随时refresh看到变化。
